Welcome to the rebootPianoteq4 GitHub Project Page.

This project aims to provide a simple & automated way to handle tedious tasks necessary to effisciently use the Pianoteq4 trial ...

aka "close -> relaunch -> close trial dialog -> engage sustain pedal", & loop over these steps again every 19 minutes ( the "time lock" being of 20 minutes )

It 's intended to be run from the command line or by using the .App, docked or not, for easy & handy access to the end user.

Notes

The project can be used in two different ways:

-> launched like any other app, and closed like any other app OR by clicking on the "rotating cogwheel" in the Mac OS X OS top bar --> to do so, we'd use the .app

-> launched from the cli, and stopped/close when hitting "Ctrl-C" --> to do so, we'd use the "./stag_rebootPianoteq4.sh" command to launch the bash program handling necessary stuff (...)

Note that either way, once it's running, the "watching process", while in use, displays a rotating cogwheel in Mac OS X top bar. If the user wants to stop it, he can either right click the icon in the dock or click the cogwheel icon in the top OS bar & selecting "stop process" BUT! -> line right below ;) If the "watching process" was launched using the current bash script and thus from the cli, it is appropriate to just "Ctrl-C" the script & let it handle the necessary "cleanup" by itself.

-> If for any reason, the bash script weren't able to stop the "watching process" as well as closing itself after the user hit "Ctrl-C", then the "watching process" 'll still be stoppable by either right clicking the icon in the dock or clicking the rotating cogwheel icon in the top OS bar & selecting "stop process", as we do while using the .app ( rebootPianoteq4.app ).

Usage

# this is the way we use the program from the cli
$ ./stag_rebootPianoteq4.sh
(..)
# to stop the background "watching process" & quit rebootPianoteq4, simply hit "Ctrl-C"
$ <Ctrl-C>
